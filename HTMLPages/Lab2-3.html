1. Define the Problem
    • This step involves gathering the requirements of the program needed to solve the problem
    • Questions that should be answered:
      1. What data do we need to process?
      2. How should we process the data?
      3. What output should the program deliver?
      4. How should the user interface look?
  2. Design the Program
    • Progran design focuses on breaking down the progarm into smaller, more manageable components. Various tools are used to help design a program.
    • Commonly used tools:
      • Structure Charts
      • Algorithm
      • Flow Charts
      • Decision Tables
      • Pseduocode
  3. Code the Program
    • Once the program has been designed, a developer then writes the code.
    • Coding often involves translating the design from charts and pseduocode into a specific programming language. Usually the language is chosen during the "defining" stage.
  4. Testing and Debugging
    • After code has been written this stage revolves around fixing any bugs in the program.
    • There are two kinds of errors that cause bugs:
      1. Syntax Errors: These are like grammatical mistakes in the code.
      2. Logical Error: These are mistakes might cause an other than intented output the code still works.
    • The programmer must fix these mistakes by carefully examining the already written code for errors.
  5. Documentation
    • After coding has been completed, the programmer must write a manyak tgat generally contains the following:
      1. An overview of the programs functionality.
      2. In-depth explanations of program features.
      3. Documentation of all commands.
      4. A thorough desciption of error messages generated by the program.
  6. Deploying and Maintaining the Program
    • A software team will be responsible foro fixing new errors and updating the software with new features.
    • When a new feature should be added to the program, the cycle restarts as developers begin to define what problem the new fature will solve.
